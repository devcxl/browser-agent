# PRD: 浮动聊天入口（页面内可拖拽按钮 + 内嵌聊天面板）

**日期:** 2026-07-19
**状态:** Confirmed
**优先级:** P1

## 1. 概述

### 1.1 问题陈述

当前唯一的聊天入口是浏览器工具栏图标打开的 Side Panel。用户在浏览网页时想要唤起 Agent 对话，必须离开当前视觉焦点去点击工具栏；Side Panel 还会挤压页面布局、且对所有标签页全局共享一个面板。

Firefox 不支持 `side_panel` API，聊天入口进一步弱化。

参考网页客服组件的形态，在每个页面内提供一个浮动的、可拖拽的圆形按钮，点击后直接在页面内展开手机大小的聊天面板，可以显著缩短「浏览 → 提问」的路径。

### 1.2 目标用户

- 浏览网页时希望随时就当前页面内容提问的现有用户
- 觉得 Side Panel 挤压布局、希望聊天面板悬浮于页面之上的用户
- Firefox 用户（无 Side Panel 入口）

### 1.3 业务价值

- 将聊天入口从浏览器 chrome 下沉到页面内，降低唤起成本
- 页面内面板与 Side Panel 共享同一套聊天实现和会话数据，体验一致
- 一套 content script 方案天然跨浏览器，顺带补齐 Firefox 的聊天入口缺口

### 1.4 成功标准

- 开启开关后，任意 http(s) 页面出现圆形浮动按钮（项目 logo），不破坏页面布局与交互
- 按钮可拖拽，松手后吸附到最近的左/右屏幕边缘，位置全局跨站点记忆
- 点击按钮在同侧滑出约 380×640 的聊天面板，面板内是完整的现有 Side Panel 聊天能力（含工具调用、Markdown、Provider 引导）
- 面板内会话与 Side Panel 的会话列表完全共享，无需刷新同步
- 提供全局开关（关闭时页面内不产生任何挂载 DOM）和站点黑名单（快捷隐藏 + 设置页管理）
- Chrome MV3 与 Firefox MV3 均通过功能和交互验收

## 2. 范围

### 2.1 In Scope

- content script 注入的浮动按钮（圆形、项目 logo、Shadow DOM 隔离）
- 按钮拖拽、左/右边缘吸附动画、全局位置持久化
- 面板容器：同侧滑出、固定尺寸（约 380×640，按视口收缩）、滑入滑出动画
- 面板内以 iframe 加载 `sidepanel.html`，完整复用现有聊天能力与会话存储
- 面板头部关闭按钮（iframe 内渲染，通知外层关闭）
- 全局开关：开启才挂载 UI；关闭时不创建任何 DOM，且已打开标签页实时生效
- 站点黑名单：按钮上右键/长按快捷「在此站点隐藏」；设置页查看、删除黑名单条目
- 设置页新增「浮动按钮」配置区（开关 + 黑名单管理）
- 新增 UI 文案接入中英文 i18n
- Chrome MV3 与 Firefox MV3 兼容

### 2.2 Out of Scope

- 面板宽高可调、尺寸记忆
- 按站点记忆按钮位置
- Esc 键关闭面板、点击面板外部自动关闭
- 按钮外观自定义（图标、颜色、大小）
- 页面内面板与 Side Panel 使用不同会话
- 移动端浏览器适配

## 3. 功能需求

### F1. 浮动按钮

- 圆形按钮，直径约 48px，内嵌扩展 logo，带阴影与悬停态。
- 使用 Shadow DOM 与页面样式隔离；`z-index` 足够高（≥ 2147483646），但不拦截按钮区域以外的页面事件。
- 默认位置：屏幕右侧、垂直居中。
- 在黑名单站点或全局开关关闭时，按钮完全不出现（不创建任何 DOM）。

### F2. 拖拽与边缘吸附

- 按住按钮可自由拖拽，拖拽过程中按钮跟随指针。
- 松手后按钮自动平滑吸附到最近的左或右屏幕边缘（水平方向），垂直位置停留在松手处，并收敛在视口内。
- 点击与拖拽通过位移阈值（约 5px）区分；拖拽不触发面板开合。

### F3. 位置持久化

- 吸附完成后将位置（边缘 + 垂直偏移）写入扩展存储。
- 所有站点共享同一位置；新页面加载时按存储位置渲染。
- 存储读取失败时回退到默认位置。

### F4. 聊天面板

- 点击按钮切换面板开合；再点一次或点击面板头部关闭按钮关闭。
- 面板从按钮所在边缘同侧滑出，宽约 380px、高约 640px；视口不足时收缩并保留安全边距。
- 面板带圆角、阴影与滑入/滑出动画。
- 点击页面其他区域不自动关闭面板；面板打开期间按钮保持可见可用。

### F5. 完整 Agent 能力

- 面板内通过 iframe 加载扩展自身的 `sidepanel.html`。
- 聊天能力、工具调用确认、Provider 配置引导、主题、语言与 Side Panel 完全一致。
- 会话数据与 Side Panel 共享同一会话存储；在任一入口发送的消息在另一入口可见（重新打开入口后即可见，不要求实时跨入口推送）。
- iframe 以查询参数标记内嵌模式（如 `?embedded=1`），供 Side Panel 渲染面板头部关闭按钮等内嵌差异。

### F6. 全局开关

- 设置页提供「浮动按钮」总开关。
- 开关关闭时，content script 启动直接退出，不创建按钮、面板、监听器以外的任何 DOM。
- 切换开关时，所有已打开标签页无需刷新立即挂载/卸载按钮。

### F7. 站点黑名单

- 在按钮上右键（或长按）弹出小型菜单，提供「在此站点隐藏」；确认后当前站点域名加入黑名单，按钮立即消失。
- 黑名单按站点主机名匹配（含其子域名）。
- 设置页黑名单列表可查看条目并逐个移除；移除后该站点下次加载恢复显示按钮。

### F8. 设置页管理

- 设置面板新增「浮动按钮」区块：总开关、黑名单列表管理。
- 该区块不改变现有设置项的结构与行为。

### F9. 国际化

- 按钮无障碍名称、黑名单菜单、设置区块文案提供中文和英文。
- 语言切换后新增文案与应用当前语言一致。

### F10. 跨浏览器

- Chrome MV3 与 Firefox MV3 使用同一实现。
- Firefox 上浮动按钮作为聊天主入口之一；现有各浏览器入口行为不变。

## 4. 用户故事与验收标准

### US-1: 页面内唤起聊天

**作为** 正在浏览网页的用户
**我想要** 点击页面内的浮动按钮直接开始对话
**以便** 不用去工具栏找入口，也不挤压页面布局

**验收标准：**

- [ ] 开启开关后页面出现圆形 logo 按钮，默认位于右侧垂直居中
- [ ] 点击按钮从同侧滑出约 380×640 面板，iframe 内为完整聊天界面
- [ ] 面板内可发送消息、触发工具调用、渲染 Markdown，与 Side Panel 行为一致
- [ ] 未配置完整 Provider 时，面板内同样出现配置引导
- [ ] 再点按钮或点面板头部关闭按钮，面板滑出关闭
- [ ] 面板内发送的消息，重新打开 Side Panel 后出现在同一会话中

### US-2: 拖拽且不碍事

**作为** 用户
**我想要** 把按钮拖到屏幕任意一侧
**以便** 它不遮挡我关注的内容

**验收标准：**

- [ ] 按住拖拽时按钮跟随指针，松手后平滑吸附到最近的左/右边缘
- [ ] 位移小于约 5px 视为点击，不误触发拖拽
- [ ] 吸附后位置写入存储，其他站点和重启浏览器后保持一致
- [ ] 按钮在任何站点不破坏页面原有布局与点击交互

### US-3: 防打扰控制

**作为** 用户
**我想要** 在某些站点或全局关掉浮动按钮
**以便** 阅读和工作时不受干扰

**验收标准：**

- [ ] 设置页总开关关闭后，所有已打开标签页按钮立即消失且无残留 DOM
- [ ] 重新打开开关后，已打开标签页按钮无需刷新立即出现
- [ ] 按钮右键/长按菜单选择「在此站点隐藏」后，该站点按钮立即消失
- [ ] 黑名单站点重新加载页面后仍不显示按钮
- [ ] 设置页可查看黑名单并移除条目，移除后该站点恢复显示

## 5. 非功能需求

### 5.1 性能

- 开关关闭或命中黑名单时，content script 除一次存储读取外零开销。
- 面板未打开时不加载 iframe（懒加载，`sidepanel.html` 仅在首次打开时加载）。
- 拖拽与吸附动画不得引起页面重排；仅使用 transform/opacity。

### 5.2 兼容性

- Chrome MV3 与 Firefox MV3 均通过构建与交互验证。
- `sidepanel.html` 需声明为 web accessible resource，且不得因此暴露其他扩展内部页面。

### 5.3 安全与隐私

- 按钮与面板不读取、不修改页面 DOM 与数据；黑名单仅存站点主机名。
- iframe 仅加载扩展自身页面，不接受页面脚本传入的消息指令。
- 父页与 iframe 间的消息通信校验来源与消息类型白名单。

### 5.4 可访问性

- 按钮提供可访问名称与键盘可达性（聚焦后 Enter/Space 开合面板）。

## 6. 约束与依赖

- 基于现有 WXT + React + Tailwind 体系；不新增 npm 依赖。
- 复用 `sidepanel.html` 全部现有实现，不复制聊天逻辑。
- 存储层扩展 `StorageSchema`，沿用 `ConfigStore` 与现有迁移机制。
- 主要相关文件：
  - `src/content/index.ts`（content script 入口）
  - `src/entrypoints/sidepanel/App.tsx`（内嵌模式差异）
  - `src/entrypoints/sidepanel/components/SettingsPanel.tsx`（设置区块）
  - `src/shared/types/storage.ts`、`src/shared/storage/config-store.ts`（存储扩展）
  - `wxt.config.ts`（web accessible resources 声明）

## 7. 验收环境

- 浏览器：Chrome MV3、Firefox MV3
- 站点类型：普通站点、严格 CSP 站点（如 GitHub）、黑名单站点
- 视口：1366×768、1920×1080；面板在小窗口下的收缩表现
- 主题：Light、Dark、System
- 语言：中文、英文
- 状态：开关开/关、黑名单有/无、Provider 已配置/未配置
